# 티웨이브 채용 지원: 분석 및 맞춤 이력서

---

## 1. 티웨이브(주) 분석

### 1.1 회사 개요
- **정체성**: 핀테크와 결합한 혁신적인 **소셜 네트워크(Social Network)** 를 만드는 기업
- **대표 서비스**: **아임인(imin)** — 전국민 목돈 모으기 실천 앱, 목돈마련·합리적 소비를 지원하는 **라이프스타일 핀테크**
- **비전**: 금융의 민주화, 신용 기반의 강력한 네트워크 구축, “유저 간 신뢰와 관계 형성” 중심 플랫폼

### 1.2 기술·운영 특징 (창업스토리·매체 기사 기반)
- **데이터 일관성·트랜잭션 안정성**: 다중 트랜잭션 환경에서 시스템 안정성 유지, 고급 DB 관리·실시간 모니터링
- **자체 신용 평가**: 연체 가능성 사전 감지, 어뷰징 방지
- **보안**: VPC(가상 사설 클라우드), 통신 암호화, DB 중요 정보 암호화 저장, WAF 도입
- **미래 방향**: AI 등 최신 기술 도입, 사용자 경험 최우선

### 1.3 채용 포지션과의 연결
- “**미래 10년을 책임질 차세대 소셜 핀테크**” → 결제/출금/정산·신용·트랜잭션 안정성 경험이 직결
- “**직접 설계하고 직접 코드까지 만드는**” 시니어 → 설계와 구현을 모두 수행한 경력을 강조하는 것이 유리

## 2. 채용 공고 상세 분석

**포지션**: 시니어 백엔드 개발자 (8년 이상 ~ 20년 이하)  
**근무지**: 서울 강남구 삼성로85길 26  
**고용형태**: 정규직 (상시 채용)

### 2.1 기대 역할
- **실무형 리드**: “단순한 설계자나 리뷰어가 아닌, **직접 설계하고 직접 손으로 코드까지 만드는** 엔지니어링 중심의 아키텍트”
- 프로젝트 **핵심 도메인 아키텍처**를 주도적으로 설계하고, **최신 기술 스택**으로 **핵심 코드를 직접 구현**

### 2.2 주요 업무 (요약)

| 비중 | 영역 | 세부 내용 |
|------|------|-----------|
| **약 70%** | 서비스 아키텍처 설계 + 핵심 코드 구현 | ① **핀테크 Core 비즈니스**: imin 리빌드 — 스테이지, 결제/출금/정산 등 도메인 로직 설계·구현 (Java/Kotlin, Spring Boot), 복잡한 트랜잭션·데이터 정합성, MSA 또는 DDD 기반 확장 가능 구조, 핵심 API 개발 ② **레거시 현대화·리팩토링 주도**: 레거시 분석 → 현대적 구조로 전환·마이그레이션 직접 구현, 코드 품질·성능 개선, 기술 부채 해소 ③ **분산 시스템·DB**: 대량 트랜잭션 처리용 분산 시스템(Kafka, Redis 등) 설계·구현, DB 스키마 설계, JPA/Hibernate 기반 쿼리 성능 튜닝 |
| **20~30%** | 기술 리더십·표준 수립 | 아키텍처·기술 스택 의사결정 주도, 직접 작성한 코드를 통해 설계 노하우·방향성 공유 |

### 2.3 핵심 키워드
- **Java / Kotlin / Spring Boot**, **핀테크 Core**(결제·출금·정산), **트랜잭션·데이터 정합성**
- **레거시 현대화·마이그레이션**, **MSA·DDD**, **분산 시스템**(Kafka, Redis), **DB 스키마·JPA/Hibernate 성능**
- **직접 설계·직접 구현**, **기술 리더십**

### 4.3 자기소개서 (티웨이브·아임인 맞춤)

#### (1) 지원 동기 및 티웨이브/아임인 이해

티웨이브는 “핀테크와 결합한 소셜 네트워크”로, 아임인을 통해 목돈마련과 합리적 소비를 지원하면서 **신용 기반의 관계와 플랫폼**을 지향한다고 이해했습니다. 금융의 민주화와 사용자 간 신뢰를 기술로 구현하는 방향성이 제가 페이레터·핀업에서 쌓아온 **결제·정산·트랜잭션 안정성**과 **사용자 중심 서비스** 경험과 잘 맞는다고 생각해 지원하게 되었습니다.

또한 채용공고에서 말씀하신 “**직접 설계하고 직접 손으로 코드까지 만드는**” 시니어 역할에 공감합니다. 저는 20년 이상된 레거시 결제 시스템의 마이그레이션을 기획·설계부터 구현·배포까지 주도했고, 다양한 결제·정산·AML 기능을 설계와 코드 작성 모두 담당해왔습니다. 설계만 하거나 리뷰만 하는 역할이 아니라, **아키텍처를 정하고 그에 맞는 핵심 코드를 직접 작성하는** 방식으로 일해왔으며, 티웨이브의 imin 리빌드와 핵심 도메인(스테이지, 결제/출금/정산) 구현에 그 경험을 기여하고 싶습니다.

#### (2) 핀테크 Core·트랜잭션·데이터 정합성 경험

페이레터에서 **결제·출금·정산**과 **자금세탁방지(AML)** 까지 담당하며, “한 번의 오류도 허용하기 어려운” 환경에서 **트랜잭션 일관성과 데이터 정합성**을 최우선으로 개발해왔습니다. 정산·통계용 DB를 분리 설계하고, 프로시저와 트랜잭션 관리로 대용량 처리와 안정성을 확보한 경험이 있습니다. 티웨이브가 강조하는 **복잡한 트랜잭션·데이터 정합성** 요구에, 제가 쌓은 결제·정산 도메인 경험으로 바로 기여할 수 있다고 생각합니다.

#### (3) 레거시 현대화·리팩토링 주도

20년 이상 운영된 Classic ASP 기반 결제 시스템을 **.NET Core 기반으로 전면 마이그레이션**한 경험이 있습니다. 기존 코드와 비즈니스 로직 전수 분석, 새 아키텍처 설계, 비동기 처리·Connection Pooling·JWT 인증·EF Core 도입을 직접 구현했고, 그 결과 모니터링 부담 감소와 시스템 안정성·유지보수성 개선을 이루었습니다. “레거시를 현대적 구조로 전환하고, 기술 부채를 해소하는” 업무를 **설계와 구현 모두 주도**해온 점이, imin 리빌드와 레거시 현대화가 중요한 티웨이브에 잘 맞을 것으로 봅니다.

#### (4) 기술 스택·학습 의지

현재 실무에서는 주로 **C# / .NET Core / MS-SQL**로 개발해왔고, 이력서에 기재한 대로 **Java, Spring Boot**도 보유하고 있습니다. 채용공고의 **Java/Kotlin, Spring Boot** 기반 개발에 맞춰, 기존에 쌓은 **트랜잭션 설계·API 설계·DB 설계·성능 튜닝** 경험을 빠르게 적용할 수 있다고 생각하며, 필요 시 **Kafka, Redis, JPA/Hibernate** 등 분산·DB 스택도 업무에 맞춰 학습·도입할 각오가 되어 있습니다.

#### (5) 마무리

15년간 결제·정산·레거시 현대화를 **직접 설계하고 직접 구현**해온 백엔드 개발자로서, 티웨이브의 “미래 10년을 책임질 차세대 소셜 핀테크”에 **아키텍트이자 구현자**로 참여하고 싶습니다. 아임인 리빌드와 핵심 도메인 안정성·확장성에 기여할 수 있도록 지원합니다.
