자기소개서 (티웨이브·아임인 지원)
1. 티웨이브에 지원하는 이유 — 회사와 제 경험의 접점

티웨이브는 전통적인 ‘계모임’을 디지털화하여, 지인 간 신뢰에 기반한 새로운 형태의 P2P 금융을 만드는 회사로 이해했습니다. 아임인은 사람들이 함께 돈을 모으고 매달 정한 순서대로 목돈을 마련하는 서비스이고, 스테이지(계)는 앞순번의 목돈 대출부터 후순번의 재테크까지 각자의 니즈를 맞추는 구조입니다. 0.3%라는 시중 은행(0.5%)보다 낮은 연체율과, 커뮤니티·트립·커머스·B2B로의 확장은 이미 검증된 비즈니스 모델과 기술의 안정성을 보여준다고 봅니다. 매출 44억→80억→약 150억으로의 성장, 3년 연속 흑자, 누적 투자유치 129억은 그 결과라고 생각합니다.

저는 페이레터에서 결제·출금·정산·자금세탁방지(AML) 시스템을 설계·개발·운영했고, 핀업에서는 핀테크 스타트업의 초창기 멤버로 조직 15명→50명, 연매출 50억→200억 성장 과정에 참여했습니다. “한 번의 오류도 허용하기 어려운” 금융 도메인에서 트랜잭션 일관성과 데이터 정합성을 최우선으로 개발해왔고, 스타트업 성장기에 서비스를 안정적으로 지탱하는 일을 경험했습니다. 티웨이브의 성장 궤적과 제가 쌓아온 경험이 잘 맞는다고 판단해 지원했습니다.

회사의 성장과 제 성장이 함께해온 경험이 있습니다. 핀업에서 초창기 15명일 때 입사해, 서비스 런칭과 안정화를 거쳐 50명 이상·연매출 200억원 규모로 성장하는 과정을 같이했습니다. 그때 배운 것은 “빠른 성장 속에서도 안정성을 지키는 것”의 균형이었고, 기술 부채가 쌓일 때 리뉴얼과 아키텍처 일원화로 함께 맞서본 경험이 있습니다. 티웨이브도 33명 규모에서 매출 150억을 넘어가며, 아임인트립·아임인 파트너스·커머스·B2B로 영역을 넓혀가고 있습니다. 저는 이다음 단계—MSA 전환, 분산 처리, 대규모 트래픽—에서 티웨이브의 성장과 함께, 제가 새로운 도전(Java/Kotlin, Kafka, Redis, DDD 등)을 배우며 성장하는 기회를 갖고 싶습니다. 함께 잘 커가는 조직에서 기여하고 성장하는 것을 중요하게 생각합니다.

2. Legacy → Modern 리빌드, 직접 설계·직접 구현 경험

채용공고에서 “Legacy to Modern 리빌드 주도”, “기존 모놀리식 비즈니스 로직을 DDD 기반 MSA 구조로 마이그레이션”하시는 분을 찾고 계시는 것으로 이해했습니다. 저는 20년 이상 운영된 Classic ASP 기반 결제 시스템을 .NET Core로 전면 마이그레이션한 프로젝트를 기획·분석·아키텍처 설계·구현·배포까지 직접 주도했습니다. 단순 코드 변환이 아니라 기존 비즈니스 로직 전수 분석, 새 아키텍처 설계, 비동기 처리·Connection Pooling·JWT·EF Core 도입을 직접 구현했고, 그 결과 모니터링 부담 감소와 시스템 안정성·유지보수성 개선을 이루었습니다. “설계자나 리뷰어가 아닌, 직접 설계하고 직접 코드까지 만드는” 역할을 해왔으며, 아임인 리빌드와 Legacy → Modern 전환에 이 경험을 기여하고 싶습니다.

3. 핀테크 도메인 — 정산·결제·원장 시스템 경험

채용공고의 “핀테크/금융 도메인의 원장(Ledger), 정산, 결제 시스템을 깊이 있게 다뤄본 경험”을 우대하신다고 했습니다. 저는 페이레터에서 결제·출금·정산 전반과 AML 시스템(가맹점 정보 갱신 자동화, 위험군 식별 로직)을 설계·구현했고, 정산·통계용 DB를 분리 설계해 대용량 데이터 처리 성능을 높였습니다. MS-SQL 프로시저·트랜잭션·데이터 모델링 다수를 수행했고, 24/7 결제 시스템 모니터링과 장애 대응을 경험했습니다. 스테이지의 순번별 입출금·정산·원금 보장 로직이 요구하는 트랜잭션 처리와 데이터 정합성에, 제가 쌓은 결제·정산 도메인 경험을 적용할 수 있다고 생각합니다.

4. 고가용성·성능 최적화·기술 학습 의지

“초당 수만 건의 트래픽이 몰려도 흔들리지 않도록 Kafka, Redis 등을 활용한 분산 처리”를 구현하시는 일에 관심이 있습니다. 저는 정산·통계 DB 분리 설계, 쿼리·인덱스 튜닝, 프로시저 최적화로 대용량 처리 성능을 개선한 경험이 있습니다. 현재 실무는 C#/.NET Core/MS-SQL 기반이지만, Java, Spring Boot도 보유하고 있으며 Kotlin은 아직 경험이 없습니다. 채용공고에서 “언어의 문법보다 아키텍처를 보는 눈과 문제 해결 능력이 중요하다”고 하신 점에 공감합니다. 기존의 트랜잭션 설계·API 설계·DB 설계·성능 튜닝 경험을 바탕으로 Java/Kotlin Spring Boot 기반 개발에 빠르게 적응하고, Kafka, Redis, JPA/Hibernate 등 필요 스택도 업무에 맞춰 학습·도입할 준비가 되어 있습니다.

5. 테스트·코드 품질·주니어와의 협업

“JUnit, Kotest 등을 활용한 테스트 작성 생활화”, “주니어 개발자들에게 Clean Code와 TDD 문화를 전파”하시는 역량을 기대하신다고 이해했습니다. 저는 스트롱홀드에서 팀장·과장으로 소규모 팀을 이끌며 프로젝트 일정·품질 관리, 개발 프로세스 개선을 경험했습니다. 실제 TDD 정착 경험은 제한적이지만, 예외 처리·에러 핸들링·로깅·모니터링 등 운영 관점의 코드 품질을 중시해왔고, 주니어에게 아키텍처와 설계 관점의 인사이트를 나누는 데 관심이 있습니다. “나이/연차 상관없이 실력적으로 뛰어나신 분”, “IC Track 경험이 있는 시니어”를 선호하신다는 점과도 맞닿아 있다고 생각합니다.

6. 마무리

15년간 결제·정산·레거시 현대화를 직접 설계하고 직접 구현해온 백엔드 개발자로서, 티웨이브의 “미래 10년을 책임질 차세대 소셜 핀테크”에 참여하고 싶습니다. 아임인 리빌드와 스테이지·결제·출금·정산 등 핵심 도메인의 안정성·확장성에 기여할 수 있도록 지원합니다. 서울 강남구 대치동 근무지도 희망 근무 지역과 일치합니다. 감사합니다.
