#### Q1. Java 경험이 많지 않은데, 어떻게 생각하시나요?

**답변**:
> "15년 6개월간 C#/.NET을 주력으로 사용해왔지만, 최근 Java/Spring Boot를 학습하여 토론 플랫폼 팀 프로젝트를 완성했습니다. Java와 C#은 동일한 객체지향 패러다임을 공유하며, 문법도 매우 유사합니다.
> 
> 제 강점은 **특정 언어에 구애받지 않고 시스템의 본질을 이해하고 문제를 해결하는 능력**입니다. Classic ASP에서 .NET Core로 마이그레이션 프로젝트를 주도하며, 단순히 언어를 바꾸는 것이 아니라 시스템 아키텍처를 개선하고 성능을 최적화하는 경험을 쌓았습니다.
> 
> Java 문법은 빠르게 익힐 수 있으며, 오히려 **15년간 축적한 백엔드 시스템 설계, 데이터베이스 최적화, 실시간 처리 경험**이 와따 에이아이의 AI 플랫폼 개발에 더 큰 가치를 제공할 수 있다고 확신합니다."

**핵심 메시지**:
- Java는 도구일 뿐, 시스템 설계 능력이 핵심
- 빠른 학습 능력 입증 (Spring Boot 프로젝트)
- 15년 백엔드 경험의 가치 강조

#### Q2. 대용량 실시간 데이터를 처리한 경험이 있나요?

**답변**:
> "페이레터에서 2년 9개월간 대규모 트래픽 환경의 결제 시스템을 운영했습니다. 특히 아프리카TV와 같은 대형 클라이언트의 결제를 처리하며, 실시간 트랜잭션 처리의 중요성을 체감했습니다.
> 
> **구체적인 경험**:
> 
> 1. **비동기 처리 및 Connection Pooling**
>    - Classic ASP의 DB 연결 비효율성 문제를 .NET Core의 비동기 처리와 Connection Pooling으로 해결
>    - 동시 연결 수 증가 및 응답 시간 개선
> 
> 2. **DB 분리 설계**
>    - 정산 및 통계 DB를 분리하여 대용량 데이터 처리 성능을 향상
>    - 운영 DB와 분석 DB 분리로 서비스 안정성 확보
> 
> 3. **실시간 모니터링**
>    - 잠재적 오류(ASP_0147, 0148)를 사전에 감지하고 해결하는 시스템 구축
>    - 24/7 모니터링 및 장애 대응 프로세스 수립
> 
> 와따 에이아이의 3D LiDAR 데이터 처리도 유사한 접근이 필요하다고 생각합니다. 대량의 센서 데이터를 실시간으로 수집하고, 효율적으로 저장하며, 빠르게 조회하는 시스템 설계 경험이 직접적으로 도움이 될 것입니다."

**핵심 메시지**:
- 실시간 트랜잭션 처리 경험 (결제 시스템)
- 구체적인 성능 개선 사례
- 와따 에이아이 시스템과의 유사성 연결

#### Q3. RESTful API 설계 경험을 설명해주세요.

**답변**:
> "15년간 다양한 API를 설계하고 개발해왔습니다. 특히 세 가지 프로젝트가 대표적입니다:
> 
> **1. 페이레터 - 결제 API 리팩토링**
> - URL 결제, 헥토펌뱅, 애플페이 등 다양한 결제 방식을 통합하는 REST API 설계
> - JWT 토큰 기반 인증 시스템 구축
> - Parameterized Query로 SQL Injection 방지
> - 각 결제 방식별 프로토콜을 통합하여 **일관된 인터페이스** 제공
> 
> **2. 핀업 - 통합 플랫폼 API**
> - PC, 모바일, 관리자 시스템이 하나의 API를 사용하도록 설계
> - OAuth2 인증 흐름 구현 (Apple SNS 연동 포함)
> - 사용자 리포트, 랭킹 시스템 API 개발
> 
> **3. 토론 플랫폼 (Spring Boot)**
> - RESTful 원칙에 따른 API 설계
> - WebSocket과 REST API의 하이브리드 구조
> - JWT 기반 인증 및 권한 관리
> 
> **설계 원칙**:
> - 명확한 엔드포인트 네이밍 및 HTTP 메서드 활용
> - 버전 관리 전략 (하위 호환성 유지)
> - 에러 핸들링 및 일관된 응답 포맷
> - API 문서화"

**핵심 메시지**:
- 다양한 프로젝트에서 API 설계 경험
- RESTful 원칙 이해
- 인증/보안 고려
