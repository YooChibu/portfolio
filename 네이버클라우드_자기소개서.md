# 네이버 클라우드 API Gateway 경량화 연구 개발 - 자기소개서

## 1. 자기자신을 자유롭게 소개해 주세요. (최대 1,000자)

안녕하세요. 15년 6개월간 웹 및 백엔드 개발자로 근무한 유치부입니다.

**가치관과 성격의 장점:**
저는 '신뢰는 성실함에서 비롯된다'는 신조를 가지고 있습니다. 맡은 일은 끝까지 책임지고 완수하며, 복잡한 시스템 구조 속에서도 문제의 본질을 파악하고 해결책을 찾아내기 위해 끝까지 집중하는 것이 저의 강점입니다. 또한 안정성과 유지보수성을 최우선으로 고려하는 개발 철학을 가지고 있어, 단기적인 해결책보다는 장기적인 관점에서 시스템을 설계하고 개선합니다.

**성격의 단점과 극복 노력:**
신중한 성향으로 인해 업무 시간이 다소 늘어나는 경우도 있지만, 이는 서비스의 안정성과 완성도를 높이기 위한 고민의 결과입니다. 결제 시스템과 같이 실시간으로 대량의 트래픽을 처리해야 하는 서비스를 운영하면서, 작은 실수도 큰 문제로 이어질 수 있다는 것을 체감했습니다. 따라서 코드를 작성할 때 단순히 동작하는 것에 그치지 않고, 예외 상황 처리, 에러 핸들링, 로깅, 모니터링 등 운영 관점에서의 고려사항까지 함께 생각하며 개발합니다.

**경력:**
지난 15년 6개월간 결제 시스템, 교육 플랫폼, 투자 콘텐츠 서비스 등 다양한 도메인을 경험했습니다. 특히 페이레터에서 20년 이상 운영된 Classic ASP 기반 결제 시스템을 .NET Core로 마이그레이션하는 프로젝트를 주도하며, 단순히 기술을 교체하는 것이 아니라 시스템의 본질을 이해하고 개선하는 것의 중요성을 깊이 체감했습니다. 최근에는 새로운 기술 학습에도 적극적이며, Cursor AI를 활용한 프로젝트 개발, Spring Boot 기반 토론 플랫폼 개발 등을 통해 지속적인 성장을 추구하고 있습니다.

**지원동기:**
네이버 클라우드의 API Gateway 경량화 프로젝트에 지원한 이유는, 제가 경험한 시스템 리팩토링과 아키텍처 개선 경험이 이 포지션의 핵심 업무와 밀접하게 연관되어 있다고 판단했기 때문입니다. 대규모 트래픽 환경에서의 안정적인 서비스 운영 경험과 시스템 경량화를 통한 성능 개선 경험을 바탕으로, NAVER Cloud의 글로벌 서비스 강화에 기여하고 싶습니다.

## 2. 지원 직무와 관련된 자신의 대표적인 프로젝트 한 가지를 적어 주시되, 어떠한 고민을 가지고 어떻게 결과물을 만들어 냈는지, 기여도는 어떠했는지 구체적으로 설명해 주세요. (최대 1,000자)

**프로젝트: Classic ASP → .NET Core 결제 시스템 마이그레이션 (2024.01 - 2025.04)**

페이레터에서 20년 이상 운영된 Classic ASP 기반 결제 시스템을 .NET Core로 전면 마이그레이션하는 프로젝트를 주도했습니다. 이 프로젝트는 API Gateway 경량화와 유사한 맥락에서, 레거시 시스템의 경량화 및 현대화를 목표로 했습니다.

**주요 고민과 도전:**

1. **기존 시스템 분석의 어려움**: 20년간 누적된 코드베이스를 전수 분석하여 비즈니스 로직을 파악해야 했습니다.
2. **무중단 마이그레이션**: 실시간 결제 서비스 특성상 서비스 중단 없이 점진적 전환 전략이 필요했습니다.
3. **성능 개선**: Classic ASP의 잠재적 오류(ASP_0147, 0148)와 DB 연결 비효율성을 해결해야 했습니다.

**해결 과정:**

- 프로젝트 마이그레이션 계획을 수립하고, 기존 코드를 모듈 단위로 분석하여 .NET Core 아키텍처를 설계했습니다.
- 비동기 처리와 Connection Pooling을 도입하여 DB 연결 효율성을 향상시켰고, Entity Framework Core로 데이터 접근 계층을 모듈화했습니다.
- JWT 토큰 기반 인증 시스템을 구축하고 Parameterized Query를 적용하여 보안을 강화했습니다.

**기여도:**
프로젝트 전체를 기획하고 실행한 핵심 개발자로서, 아키텍처 설계부터 구현, 테스트, 배포까지 전 과정을 담당했습니다. 특히 기존 시스템의 문제점을 정확히 진단하고, 현대적인 기술 스택으로 해결책을 제시한 것이 핵심 기여였습니다.

**결과:**

- 시스템 안정성 대폭 개선 (모니터링 이벤트 감소)
- 유지보수성 향상 (모듈화된 코드 구조)
- 보안 강화 (JWT 인증, SQL Injection 방지)
- 성능 최적화 (Connection Pooling으로 DB 연결 효율성 향상)

이 경험을 통해 대규모 시스템의 경량화와 현대화에 대한 실무 역량을 확보했으며, 이러한 경험이 NAVER Cloud의 API Gateway 경량화 프로젝트에 직접적으로 기여할 수 있다고 확신합니다.

## 3. 팀을 이뤄 협업을 하면서 성과를 이뤘던 경험을 적어주세요. 자신이 어떤 역할로 팀에 기여했고 어떤 교훈을 얻었는지를 포함해 주세요. (최대 1,000자)

**경험: 핀업 초창기 멤버로서의 조직 성장 기여 (2017.06 - 2022.07)**

핀업에 초창기 멤버(15명)로 입사하여 개발본부에서 5년 2개월간 근무하며, 조직이 50명 이상으로 확장되고 연매출이 50억원에서 200억원(4배 증가)으로 성장하는 과정에 직접 참여했습니다.

**팀 내 역할과 기여:**

1. **핵심 서비스 개발 및 안정화**: 슈퍼 스탁 K2~K7, 슬기로운 투자생활, 투자 아카데미 등 핵심 서비스를 개발하고 초기 런칭 및 안정화에 기여했습니다. 초창기 환경에서 빠른 기능 개발과 버그 수정을 통해 서비스 품질을 확보했습니다.

2. **통합 플랫폼 아키텍처 설계**: PC, 모바일, API 구조를 일원화하는 서비스 리뉴얼 프로젝트에서 아키텍처 설계를 주도했습니다. 개발팀, 디자인팀, 기획팀과 협업하며 통합 플랫폼을 구축하여 개발 및 유지보수 효율성을 크게 향상시켰습니다.

3. **장기 운영 및 성능 최적화**: 5년 이상 장기 운영하며 지속적인 성능 개선과 기능 고도화를 주도했습니다. 사용자 증가에 따른 시스템 부하 문제를 해결하고, 랭킹 알고리즘 최적화 등을 통해 서비스 안정성을 확보했습니다.

**주요 성과:**

- 조직 규모 15명에서 50명 이상으로 확장되는 성장 과정에 핵심 개발자로서 기여
- 연매출 50억원에서 200억원으로 4배 성장에 기여
- 통합 플랫폼 개발로 서비스 일원화 및 운영 효율성 향상
- 다양한 서비스(슈퍼스탁, 슬기로운 투자생활, 챌린지) 통합 개발

**얻은 교훈:**

1. **성장하는 조직에서의 적응력**: 조직이 빠르게 성장하는 과정에서 팀원이 늘어나고 업무 프로세스가 변화하는 환경에 적응하며, 새로운 팀원들과의 협업 방식을 지속적으로 개선해야 함을 배웠습니다.

2. **장기적 관점의 중요성**: 초기에는 빠른 기능 개발이 중요했지만, 시간이 지나면서 기술 부채가 누적되는 문제를 경험했습니다. 이후 서비스 리뉴얼을 통해 통합 플랫폼을 구축하며, 단기적 속도보다 장기적 관점의 중요성을 깊이 체감했습니다.

3. **협업과 소통의 가치**: 개발팀뿐만 아니라 디자인팀, 기획팀과의 원활한 소통을 통해 더 나은 결과물을 만들어낼 수 있었습니다. 각 팀의 전문성을 존중하고 협력하는 것이 프로젝트 성공의 핵심임을 배웠습니다.

이 경험을 통해 성장하는 조직에서 팀과 함께 성과를 내는 능력을 확보했으며, NAVER Cloud의 다양한 팀과의 협업에서도 이러한 경험을 바탕으로 기여할 수 있을 것입니다.
